# 🚂 Deploy ke Railway - Setup Kledo Integration

## Langkah Setup Kledo di Railway

### 1. Buka Railway Dashboard
- Login ke [railway.app](https://railway.app)
- Pilih project kamu

### 2. Set Environment Variables
Klik **Variables** di sidebar, lalu tambahkan:

```
KLEDO_TOKEN = <token API dari Kledo>
```

**Cara dapat token Kledo:**
- Login ke [kledo.com](https://kledo.com)
- Menu: **Pengaturan** → **API**
- Copy **Personal Access Token**

### 3. Redeploy Service
- Setelah set `KLEDO_TOKEN`, klik **Deploy** untuk restart service
- Atau jika pakai Railway CLI: `railway up`

### 4. Verifikasi Koneksi
Setelah deploy selesai, buka:
```
https://<domain-railway-mu>/api/kledo/token-status
```

Response yang benar:
```json
{
  "valid": true,
  "status": "Token valid",
  "message": "Berhasil terhubung ke Kledo. Total invoice di akun: 123"
}
```

### 5. Jika Masih Error
- Cek log Railway untuk error message
- Pastikan token tidak expired
- Pastikan token dari akun Kledo yang benar

## Environment Variables Wajib

| Variable | Deskripsi | Contoh |
|----------|-----------|--------|
| `KLEDO_TOKEN` | Token API Kledo | `abcd1234...` |
| `DATABASE_URL` | Auto-generated by Railway | `postgresql://...` |
| `ADMIN_PASSWORD` | Password admin ERP | `admin123` |

## Troubleshooting

### Error: "KLEDO_TOKEN belum dikonfigurasi"
- Pastikan sudah set di Railway Variables
- Redeploy setelah set

### Error: "Token ditolak (HTTP 401)"
- Token salah atau expired
- Buat token baru di Kledo

### Error: "Tidak bisa terhubung ke Kledo"
- Cek koneksi internet Railway
- Cek apakah Kledo API down

## Testing Endpoint

Setelah setup benar, test endpoint berikut:

- `GET /api/kledo/token-status` - Cek status token
- `GET /api/kledo/products` - Ambil produk dari Kledo
- `GET /api/kledo/contacts` - Ambil kontak dari Kledo
- `POST /api/kledo/sync` - Sinkronisasi data penjualan
